Sweet Links

A little jQuery plugin that can add some cool attributes and features to anchor tags on your website.

What does Sweet Links do?

A few things! Sweet Links can be applied to any or all anchor tags on your website. It will check the properties of the tag, and then execute the following functions:

  • The script will check if the href points to a downloadable file format, and if so, it will add a class that can be used for styling as well as the download attribute if it doesn't already exist.
  • The script will check if the href points to an anchor on the same page, and if so, smooth scroll to that anchor. The speed of this animation and the position that the user is taken to can be configured to suit your requirements. A class is also applied which can be used for styling.
  • The script will check if there is a title attribute present on the anchor tag, and if not, it will add it using the text content of the link.
  • The script will check if the href points to an external destination. If so, it will add a class that can be used for styling, the rel="external" attribute will be added, and if desired - an optional target="_blank" (disabled by default) will be added.
  • The script will check if the href points to an email address, add a class if so, and add a title based on the email address itself, if one does not exist.

The file is 2.5kb unminified and 1.5kb minified.

Documentation

Sweet Links requires jQuery, so make sure this is referenced before Sweet Links. A version that does not require jQuery is in the works.

To add Sweet Links to your site, simply reference it after jQuery and then initialize it after document ready, as demonstrated below:

<script type="text/javascript" src="~path~/jquery.js"></script>
<script type="text/javascript" src="~path~/sweet-links.js"></script>
<script type="text/javascript">
	jQuery(document).ready(function($) {
		$("a").sweetLinks();
	});	
</script>

Simply using "<a>" as your selector will target all links on the page, so take that into account as you may want to only use the plugin on links within certain text-based areas of your site, or exclude anchors with certain classes using the :not(~) selector - the choice is yours.

Options & Classes

scrollTime

If a link points to an anchor on the same page (like the navigation on this page, for example), clicking on that link will smooth-scroll to that position. The scrollTime is how long this action will take in milliseconds. The default is 1000 (1000ms = 1 second), but you can change that to a time that suits your requirements, as demonstrated below.

<script type="text/javascript">
	jQuery(document).ready(function($) {
		$("a").sweetLinks({
			scrollTime: 3000
		});
	});	
</script>

scrollOffset

The smooth-scrolling action mentioned above can be offset with a pixel value. The default is 0, however, you may want to offset the smooth-scroll position to account for other elements around your destination. On this page for example, with the fixed header, I had to account for the height of that header overlapping the top of each section, so I offset the scroll position by -60, as shown below.

<script type="text/javascript">
	jQuery(document).ready(function($) {
		$("a").sweetLinks({
			scrollOffset: -60
		});
	});	
</script>

Class added: (classPrefix)smooth-scroll

fileTypes

Sweet Links will look for anchors that point to certain file types typically used to download resources. It will apply the download attribute, as well as adding a class that can be useful for styling. For example, if an anchor points to a PDF, the class "sl-pdf" will be applied. This can be useful in cases where designers / developers wish to use icon fonts or special treatments for certain link types. The classname prefix of "sl-" can be altered, as detailed below in the classPrefix section. The default file types are as follows;

"doc", "docx", "xls", "xlsx", "ppt", "pptx", "pdf", "pages", "keynote", "numbers".

This list can be restricted or added to, as demonstrated below.

<script type="text/javascript">
	// Restricted to just doc and docx filetypes.
	jQuery(document).ready(function($) {
		$("a").sweetLinks({
			fileTypes : ["doc", "docx"] 
		});
	});	

	// Filetype of "demo" added.
	jQuery(document).ready(function($) {
		$("a").sweetLinks({
			fileTypes : ["doc", "docx", "xls", "xlsx", "ppt", "pptx", "pdf", "pages", "keynote", "numbers", "demo"]
		});
	});	
</script>

Class added: (classPrefix)(value). For example: .sl-doc, .sl-pdf etc.

classPrefix

Any anchor tags targeted by Sweet Links will have a class added to them, so that any actions or behaviors can be easily styled if you wish to do that. The default prefix is "sl-" but this can be changed to whatever you like. For example, if you are using an icon font, and all of your icon classes are prefixed with "icon-" you can change Sweet Links' prefix to match that, as shown below:

<script type="text/javascript">
	jQuery(document).ready(function($) {
		$("a").sweetLinks({
			classPrefix : "icon-"
		});
	});	
</script>

newWindow & External Links

Opening external links in new tabs is controversial. Some people hate it, some people love it. I'm not here to take sides - it's an option that you can choose yourself. It is turned off by default.

<script type="text/javascript">
	jQuery(document).ready(function($) {
		$("a").sweetLinks({
			newWindow : true // false by default
		});
	});	
</script>

All outgoing links will have "(classPrefix)external" added as a class. If newWindow is set to true, an additional class of "(classPrefix)new-tab" will be added.

Browser Support

Sweet Links should work on everything, as long as Javascript is enabled.
